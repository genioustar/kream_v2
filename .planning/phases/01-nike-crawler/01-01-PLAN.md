### JS 추출 함수 (page.evaluate 에 전달, 셀렉터 상수를 인자로 받음)
```python
_JS_EXTRACT = """(args) => {
    const { cardSelector, titleSel, subtitleSel, priceSel, linkSel } = args;
    const cards = Array.from(document.querySelectorAll(cardSelector));
    return cards.map(card => {
        const titleEl    = card.querySelector(titleSel);
        const subtitleEl = card.querySelector(subtitleSel);
        const priceEl    = card.querySelector(priceSel);
        const linkEl     = card.querySelector(linkSel) || card.querySelector('a[href]');

        return {
            href:     linkEl ? linkEl.href : null,
            name:     titleEl ? titleEl.innerText.trim() : null,
            subtitle: subtitleEl ? subtitleEl.innerText.trim() : null,
            priceStr: priceEl ? priceEl.innerText.trim() : null,
            cardText: (card.innerText || '').trim(),
        };
    });
}"""
```

### 헬퍼 함수 (각 함수 단일 책임)

**`_find_card_selector(page) -> str | None`**
- adidas/crawler.py 의 `_find_card_selector` 와 동일한 진단 로직.
- CARD_SELECTORS 를 순회하며 `wait_for_selector(sel, timeout=SELECTOR_TIMEOUT_MS)` 로 시도.
- 첫 번째 성공 셀렉터를 반환. 카운트도 debug 로그.
- 전부 실패 시 `page.url`, `page.title()`, `body.innerText` 앞 `DIAG_BODY_TEXT_LEN` 자를 WARNING 로그로 출력 후 `None` 반환.

**`_scroll_to_bottom(page, card_selector) -> int`**
- naver/crawler.py 의 `_scroll_to_bottom` 패턴.
- `MAX_SCROLL_ATTEMPTS` 까지 반복:
  - `window.scrollTo(0, document.body.scrollHeight)` 실행
  - `wait_for_timeout(SCROLL_PAUSE_MS)`
  - `wait_for_load_state("networkidle", timeout=NETWORK_IDLE_TIMEOUT_MS)` 시도 (실패 시 무시)
  - `page.locator(card_selector).count()` 로 현재 카드 수 확인
  - 직전 카운트와 같으면 `no_change_streak += 1`, 다르면 0 으로 리셋
  - `no_change_streak >= NO_CHANGE_LIMIT` 이면 종료
- 최종 카드 수를 INFO 로그로 출력하고 반환.

**`_parse_price(price_str: str | None) -> int | None`**
- price_str 에서 정규식 `r'([\d,]+)'` 로 첫 매치 추출 → 콤마 제거 → `int()`.
- 실패 시 `None` 반환 (예외는 raise 하지 말 것). 실패 시 debug 로그.

**`_extract_model_name(card_text: str, href: str | None) -> str | None`**
- RESEARCH.md Pitfall 1 의 모델명 추출 전략 그대로 구현.
- 1순위: `STYLE_CODE_RE.search(card_text)` → 매치 시 group() 반환
- 2순위: `href` 가 있으면 마지막 path 세그먼트 추출 → `STYLE_CODE_RE.match(segment)` 매치 시 반환 (한국어 URL에서는 보통 매치되지 않음)
- 3순위: `card_text` 의 첫 줄(상품명) 을 fallback 으로 사용 (`card_text.splitlines()[0].strip()`). 빈 문자열이면 `None`.
- 주의: 모델명은 Kream 검색 키 → 스타일코드가 가장 안정적이지만 한국어 사이트에서는 노출 안 될 수 있음. 그래서 fallback 필요.

**`_should_exclude(card_text: str, subtitle: str | None) -> bool`**
- `card_text` 와 `subtitle` 을 합쳐 lowercase 변환.
- `EXCLUDE_KEYWORDS` 중 하나라도 포함되면 `True` 반환 (제외).
- (성인 URL 1차 + 키워드 2차 = RESEARCH.md Open Question 3 권고)

**`_extract_all_products(page, card_selector) -> list[NaverProduct]`**
- `crawled_at = datetime.now().isoformat(timespec="seconds")`
- `page.evaluate(_JS_EXTRACT, {...})` 로 카드 데이터 일괄 파싱 (셀렉터 상수들을 dict 로 전달)
- 결과 리스트 순회:
  - `_should_exclude` 면 debug 로그 후 skip
  - `name`/`priceStr`/`href` 중 하나라도 없으면 debug 로그 후 skip
  - `_parse_price` 결과가 `None` 이면 debug 로그 후 skip
  - `_extract_model_name(cardText, href)` 결과가 `None` 이면 debug 로그 후 skip
  - 모델명 중복 방지: 함수 내 `seen: set[str]` 로 동일 model_name 은 1회만 추가
  - `NaverProduct(site_name=SITE_NAME, product_name=name, model_name=mn, price=price, url=href, crawled_at=crawled_at)` 생성 후 append
  - 각 수집 항목 INFO 로그: `f"수집: {name!r} | 모델={mn} | 가격={price:,}원"`
- 반환: NaverProduct 리스트
